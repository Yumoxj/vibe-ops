# Interaction Principles

Core philosophy governing all Q&A and design presentation during vibe-design sessions.

## Asking Questions

**One question at a time.** Never batch questions. If a topic needs depth, break it into a sequence of sub-questions, each answered before the next is asked.

**Prefer structured options.** Offer multiple choice whenever possible. Open-ended questions are acceptable when options genuinely can't be enumerated. Always include a recommendation with reasoning — don't ask "what do you want?"

**Ask with a position.** State a recommendation and its rationale. Let the user challenge or confirm. Position-based questions surface disagreements faster than blank-slate ones.

**Make assumptions explicit.** When the user is vague, state your interpretation and ask for confirmation. Never guess silently.

## Exploring Solutions

**Present 2-3 alternatives with tradeoffs.** For every technical decision, show concrete options. State pros, cons, and which one is recommended. Include one minimal option.

**Strict YAGNI.** Cut features that aren't explicitly requested. If a feature seems useful but nobody asked for it, flag it separately — don't include it in the design.

**No placeholders.** TBD, TODO, "to be determined" are forbidden in design documents. Every section must have substantive content or be explicitly removed.

## Presenting Design

**Incremental validation.** Present design in small paragraphs (200-300 words). After each paragraph, confirm direction with the user before continuing. Avoid dumping the entire design at once.

**Stay flexible.** If user feedback contradicts earlier decisions, backtrack and adjust. Don't force consistency with a previous answer that the user now wants to change.
