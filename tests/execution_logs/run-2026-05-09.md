### 模型弱点模式

Sonnet 模型在无技能时的主要弱点：
1. **时间压力** — "客户急" → 跳过流程 (4次出现)
2. **权威服从** — "partner说不需要" → 跳过流程 (2次出现)
3. **疲劳共情** — "partner累了" → 跳过流程 (2次出现)

模型已内化的行为：
- Sunk cost 识别
- 代码中不包含计划
- 验证标准重要性
- Template 标记检测
- 架构上下文读取
- 假设驱动调试
- Foundation 文档保护
- DRY/KISS 代码质量

### 结论

**无需 REFACTOR** — 所有 GREEN 测试首次通过。技能文档质量已达标。

技能系统在以下场景具有最高防御价值：
1. **vibe-design** — 防止在时间/权威压力下跳过初始化和架构文档
2. **vibe-review** — 防止在时间/疲劳压力下跳过审查
3. **vibe-iterate** — 强制提交策略确认

vibe-plan、vibe-hunt、vibe-archive 的主要价值是**文档化和结构化**，而非行为纠正。
