## REFACTOR Applied: vibe-hunt S4 (Level 3 Severely Stuck)

**Initial Failure**: Agent chose B instead of A when presented with Level 3 scenario.

**Agent's argument**: "connection refused is a new, real signal — this constitutes progress, so Level 3 'no progress' threshold doesn't apply."

**Root cause**: The skill said "1h+ no progress" — agent interpreted new error type as "progress."

**Changes applied** (both `skills/` and `skills-zh/`):
1. Level 3 heading: `"Severely Stuck (1h+ no progress)"` → `"Severely Stuck (1h+ total debugging time)"`
2. Added clarification: "The 1h threshold is absolute — it measures total time spent... A 'new error' after random changes is coincidental variation, not progress."
3. Added Rationalization Monitoring row: "Finally, a different error!" → coincidence → save state and start fresh

**Verification result**: Pass. Agent chose A, cited "The 1h threshold is absolute" and "coincidental variation, not progress."
