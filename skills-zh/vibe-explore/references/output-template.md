# 输出模板

vibe-explore 文档的输出格式。

---

```markdown
# [标题]

**日期：** YYYY-MM-DD
**深度：** surface / standard / deep

## 摘要

[2-3 句话概述文档内容和存在原因。]

## [章节 1]

[按子主题（研究发现）或章节（文档）组织的主要内容。]

- 关键点及来源参考 [^1]
- 关键点及代码示例：

\```js
// 具体、可运行的示例
\```

## [章节 2]

[继续按子主题或章节组织内容。]

## 来源

### 外部来源（如适用）

[^1]: [来源标题](URL) — 一句话关键要点
[^2]: [来源标题](URL) — 一句话关键要点

### 本地来源（如适用）

| 文件 / 模块 | 分析内容 |
|------------|---------|
| `src/module-a/index.ts` | 公共 API 接口 |
| `src/module-b/config.ts` | 配置选项 |

## 待解问题

[仍不清楚的内容。如无则省略此节。]

- [问题 1]
- [问题 2]
```

---

## 说明

- 章节按主题组织——同一份文档可以混合外部研究发现和本地代码库分析
- 来源节根据实际使用调整：外部 URL、本地文件或两者兼有
- 不留占位符（TBD/TODO）——每个章节必须有实际内容
- 优先使用具体代码示例而非抽象描述
